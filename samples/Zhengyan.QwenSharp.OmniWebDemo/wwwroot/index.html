Qwen2.5-Omni realtime voice chat in the browser

Cross-platform demo. Microphone capture and playback happen in the browser, while the server exposes OpenAI-style chat completions, responses, and a websocket-based realtime API. The session settings below are sent to the server as `session.update`, including `instructions`, `temperature`, `top_p`, `max_output_tokens`, `return_audio`, `use_audio_in_video`, and `auto_respond`.
